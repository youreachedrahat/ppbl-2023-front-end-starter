# How to build always-succeeds.plutus in plu-ts

## Step by Step
To get started with plu-ts, we will follow the [Hello plu-ts tutorial in the official plu-ts documentation](https://pluts.harmoniclabs.tech/docs/examples/Hello%20World)

```bash
git clone https://github.com/HarmonicLabs/plu-ts-starter.git
cd plu-ts-starter
```
- To install plu-ts run `yarn`

- By default the template is "always succeed". But let us look into the code and try to understand a bit.

- Find contract `src/contract.ts`. Ignore or delete `src/MyDatum/` and `src/MyRedeemer` for now. Always succeed contract looks like this
```ts
import { bool, compile, makeValidator, pBool, pfn, pstruct, Script, ScriptType, V2 } from "@harmoniclabs/plu-ts";

const Data = pstruct({
    Anything: {}
});

export const contract = pfn([ Data.type, Data.type, V2.PScriptContext.type ],  bool)
(( datum, redeemer, ctx ) =>
    // always suceeds
    pBool( true )
);

export const untypedValidator = makeValidator( contract );
export const compiledContract = compile( untypedValidator );

export const script = new Script(
    ScriptType.PlutusV2,
    compiledContract
);
```

- To compile run `yarn start`

- By default you'll see the result in the terminal, as given in `index.ts`.

- You can save if you want to. Install `yarn @types/node` and change `index.ts` to
```ts
import { script } from "./contract";
import fs from 'fs';

fs.writeFile('always-succeed.plutus', JSON.stringify(script.toJson(), undefined, 2), (err: any) => {
    if (err) throw err;
    console.log("validator compiled succesfully! ğŸ‰\n");
});
```
